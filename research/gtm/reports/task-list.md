# Venture Scale — Task List

> Running list of tasks surfaced during research. Updated 2026-03-29.

## Product (Build the Alpha)
- [ ] Build minimum viable web app (PDF upload → stage select → analysis → shareable output)
- [ ] Analysis must complete in <60 seconds
- [ ] Make output shareable (clean URL or downloadable HTML)
- [ ] Decks ephemeral by default — delete after analysis
- [ ] No auth for alpha — frictionless access
- [ ] Mobile-friendly output (VCs read on phones)
- [ ] Record 30-second Loom demo with anonymized deck

## Distribution (Alpha Rollout)
- [ ] List 10-15 VCs by name for inner circle
- [ ] Write the DM/text — "Built this for deck screening, structured analysis in under a minute, try it on real decks, tell me what's missing"
- [ ] Send to inner circle (Phase 1, week 1-2)
- [ ] Ask each user for warm intros (Phase 2, week 3-4)
- [ ] Post in 1-2 VC Slack communities — EVCA, Gen Z VCs (Phase 3, week 5-6)
- [ ] "I built this" post on X (Phase 3)

## Track & Measure
- [ ] Track repeat usage (target: 5+ users analyze 3+ decks)
- [ ] Watch for unprompted sharing (target: 2+ users)
- [ ] Collect feature requests (target: 3+ specific asks)
- [ ] Note new relationships opened (target: 2+ from the tool)

## Strategic Clarifications (from founder input)
- This is a **lead gen tool first, product second.** Success = VC conversations, not revenue.
- Revenue trigger: **feature requests.** $5/mo start, grandfather early users, price up as features added.
- No feature requests = let it die naturally. The checklist IP survives regardless.
- Entry point (associate vs partner) doesn't matter — Imaan can navigate either path.
